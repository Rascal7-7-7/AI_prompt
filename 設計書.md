## 2. DBスキーマ（Prisma）

```prisma
// prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// セッション単位の学習者（匿名OK）
model Session {
  id         String   @id @default(cuid())
  createdAt  DateTime @default(now())
  updatedAt  DateTime @updatedAt

  progresses Progress[]
  quizResults QuizResult[]
}

// チャプター・レッスンの進捗
model Progress {
  id          String   @id @default(cuid())
  sessionId   String
  chapterId   String   // "chapter-01" など
  lessonId    String?  // "lesson-01-01" など（null = チャプター完了フラグ）
  completed   Boolean  @default(false)
  completedAt DateTime?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  session     Session  @relation(fields: [sessionId], references: [id])

  @@unique([sessionId, chapterId, lessonId])
  @@index([sessionId])
}

// クイズ結果
model QuizResult {
  id         String   @id @default(cuid())
  sessionId  String
  chapterId  String
  score      Int      // 正解数
  total      Int      // 問題数
  passed     Boolean  // 合格判定（score / total >= 0.7 など）
  answers    Json     // 各問題の回答記録
  createdAt  DateTime @default(now())

  session    Session  @relation(fields: [sessionId], references: [id])

  @@index([sessionId, chapterId])
}
```

## 3. コンテンツデータ型定義

```typescript
// types/chapter.ts

export type Difficulty = 'beginner' | 'intermediate' | 'advanced'

export type LessonBlock =
  | { type: 'text';       content: string }
  | { type: 'prompt';     label: string; content: string; copiable?: boolean }
  | { type: 'before_after'; before: string; after: string; explanation: string }
  | { type: 'fill_blank'; sentence: string; blank: string; choices: string[]; answer: string }
  | { type: 'image_sample'; prompt: string; imagePath: string; source: string; tool: string }
  | { type: 'compare';    taskDescription: string; claudePrompt: string; chatgptPrompt: string }

export interface Lesson {
  id: string              // "lesson-01-01"
  title: string
  estimatedMinutes: number
  blocks: LessonBlock[]
}

export interface QuizQuestion {
  id: string
  question: string
  choices: string[]
  answer: string
  explanation: string
}

export interface Chapter {
  id: string              // "chapter-01"
  number: number
  title: string
  description: string
  difficulty: Difficulty
  estimatedMinutes: number
  lessons: Lesson[]
  quiz: QuizQuestion[]
}
```

## 4. API Routes 設計

### POST `/api/chat`
Claude API のプロキシ（サンドボックス用）

```
Request:
{
  "prompt": "string",
  "systemPrompt"?: "string"
}

Response（StreamingTextResponse）:
  text/event-stream でストリーミング返却

Errors:
  400 Bad Request  — promptが空
  429 Too Many Requests — レート制限
  500 Internal Server Error
```

### GET/POST `/api/progress`

```
GET /api/progress?sessionId=xxx
Response:
{
  "completed": ["chapter-01", "chapter-01/lesson-01-01", ...],
  "quizScores": { "chapter-01": { score: 4, total: 5, passed: true } }
}

POST /api/progress
Body: { sessionId, chapterId, lessonId?, completed }
Response: { ok: true }
```

### POST `/api/quiz`

```
Request:
{
  "sessionId": "string",
  "chapterId": "string",
  "answers": [{ "questionId": string, "answer": string }]
}

Response:
{
  "score": 4,
  "total": 5,
  "passed": true,
  "results": [{ "questionId": string, "correct": boolean, "explanation": string }]
}
```

## 7. コンポーネント設計方針

- **Server Components** を基本とし、インタラクティブな部分のみ `'use client'`
- データフェッチは **Server Component** で行い、Propsでクライアントへ渡す
- shadcn/ui をベースに独自スタイルを上書き
- プロンプトブロックは常に **コピーボタン付き**
- クイズ・穴埋めは全てクライアントサイドで完結（APIコール不要）
- サンドボックスのみ `/api/chat` を叩く

## 8. セキュリティ設計

| 項目 | 対策 |
|---|---|
| APIキー露出 | サーバーサイドのみで使用、クライアントに渡さない |
| プロンプトインジェクション | `/api/chat` でシステムプロンプトを固定（上書き不可） |
| レート制限 | `/api/chat` にリクエスト制限を設ける（upstash/ratelimitなど） |
| XSS | dangerouslySetInnerHTML 不使用、テキストは全てエスケープ |
| 入力バリデーション | zod でAPIリクエストを検証 |
| .env | .gitignore に必ず含める、.env.example のみコミット |
